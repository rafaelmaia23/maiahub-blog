## Por que Playwright?

Depois de anos usando Cypress e Selenium, Playwright se tornou minha ferramenta favorita para testes E2E. Os motivos são claros:

- **Multi-browser**: Chrome, Firefox e Safari com a mesma API
- **Auto-wait**: Sem `sleep` ou waits manuais
- **Isolamento**: Cada teste roda em um contexto limpo
- **Trace viewer**: Debug visual incrível

## Configuração

A configuração é surpreendentemente simples:

```typescript
import { defineConfig } from '@playwright/test';

export default defineConfig({
  testDir: './tests',
  timeout: 30000,
  retries: 2,
  use: {
    baseURL: 'http://localhost:3000',
    trace: 'on-first-retry',
  },
});
```

## Padrões que Funcionam

### Page Object Model

Abstrair páginas em objetos torna os testes mais legíveis e manuteníveis:

```typescript
class LoginPage {
  constructor(private page: Page) {}

  async login(email: string, password: string) {
    await this.page.fill('[name=email]', email);
    await this.page.fill('[name=password]', password);
    await this.page.click('button[type=submit]');
  }
}
```

### Fixtures Customizados

Fixtures permitem compartilhar setup entre testes sem duplicação de código.

## Integração com CI

No GitHub Actions, Playwright funciona de forma confiável. O segredo é usar a imagem Docker oficial que já vem com todos os browsers instalados.

## Conclusão

Testes E2E não precisam ser frágeis ou lentos. Com as ferramentas e padrões certos, eles se tornam parte natural do fluxo de desenvolvimento.
