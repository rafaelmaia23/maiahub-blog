## Contexto

Nosso sistema de deploy estava mostrando sinais de idade. Builds lentos, deploys instáveis e falta de observabilidade tornavam cada release uma aventura. Era hora de uma mudança radical.

## Planejamento

Antes de tocar em qualquer código, passamos duas semanas mapeando:

1. **Dependências**: Quais serviços dependem do pipeline atual?
2. **Pontos de falha**: Onde as coisas quebram com mais frequência?
3. **Requisitos**: O que o novo sistema precisa garantir?

## Migração para Kubernetes

A decisão de migrar para Kubernetes não foi tomada levianamente. Avaliamos alternativas como ECS, Nomad e até soluções serverless.

```yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: api-server
spec:
  replicas: 3
  strategy:
    type: RollingUpdate
    rollingUpdate:
      maxSurge: 1
      maxUnavailable: 0
```

## Zero Downtime

O maior desafio foi garantir zero downtime durante a migração. Usamos uma estratégia de blue-green deployment com tráfego gradual.

## Resultados

Após a migração, o tempo de deploy caiu de 15 minutos para 3 minutos. A taxa de falhas em deploy caiu 90%.
